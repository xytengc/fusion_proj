# UDP数据接收验证程序 使用指南

## 程序简介

本项目提供了两个UDP数据验证程序，用于接收和验证FPGA发送的图像与雷达数据。

### 程序清单

| 程序文件 | 功能 | 核心特点 |
|---------|------|---------|
| `udp_verifier.py` | 标准验证程序 | CPU核心绑定、基础验证、详细日志 |
| `udp_verifier_advanced.py` | 高级验证程序 | 实时统计、高级对比、性能报告 |

---

## 功能特性

### 核心功能
- ✅ **CPU核心绑定**：自动绑定到固定小核(核1或核2)，降低系统开销
- ✅ **实时接收**：接收FPGA发来的UDP图像和雷达数据
- ✅ **数据解析**：自动解析图像(RGB)和雷达(AK类型)数据
- ✅ **参考对比**：与`cycle/_AKBK0.txt`和视频进行验证
- ✅ **性能监测**：
  - UDP传输速率 (Mbps)
  - 包率 (pps - packets per second)
  - 丢包率 (%)
  - 帧延迟 (ms)

### udp_verifier.py - 标准版

```
基础统计模式：
├── 帧级验证日志
│   ├── 图像匹配度
│   ├── 雷达数据对比
│   └── 参数误差分析
├── 实时速率监测
│   ├── 吞吐量 (Mbps)
│   ├── 包率 (pps)
│   └── 丢包率 (%)
└── 最终统计报告
    ├── 总帧数、包数、字节数
    ├── 平均性能
    └── 整体质量评估
```

### udp_verifier_advanced.py - 高级版

```
高级分析模式：
├── 实时滑动窗口统计
│   ├── 最近100包吞吐量
│   ├── 最近100帧延迟
│   └── 帧级性能曲线
├── 详细验证报告
│   ├── 图像相似度百分比 (%)
│   ├── 雷达参数误差统计
│   ├── 验证准确率评估
│   └── 分类错误分析
└── 性能统计矩阵
    ├── 雷达匹配/错误率
    ├── 图像匹配/错误率
    └── 整体准确率评分
```

## 快速开始

### 1. 环境要求

```bash
# 依赖包
pip install numpy opencv-python

# 可选（用于更好的CPU绑定）

```

**标准版运行：**
```bash
python3 udp_verifier.py
```

**高级版运行：**
```bash
python3 udp_verifier_advanced.py
```

程序会自动生成带时间戳的日志文件：

```bash
# 查看最新日志

# 或者查看所有日志文件
ls -lh udp_verify_*.log
```

## 输出解释

### 性能指标

| 指标 | 含义 | 目标值 | 说明 |
|------|------|-------|------|
| 吞吐量 (Mbps) | 每秒传输比特数 | >400 Mbps | 越高越好，受网络限制 |
| 包率 (pps) | 每秒包数 | >30k pps | 越高越好 |
| 丢包率 (%) | 丢失包占比 | <1% | 越低越好，无丢包最佳 |
| 延迟 (ms) | 帧到达延迟 | <20ms | 越低越好 |

### 验证指标

| 指标 | 含义 | 目标 | 说明 |
|------|------|------|------|
| 图像匹配度 | 接收图像与参考视频相似度 | >90% | 编码可能导致差异 |
| 雷达准确率 | 接收雷达与参考TXT匹配率 | >95% | 应尽可能接近100% |
| 参数误差 | 目标参数(R/V/A/P)差异 | <0.5 | 更小误差表示更好的数据完整性 |

## 故障排查

### 问题1: "连接被拒绝" 或 "地址已在使用"

**原因**: 端口被占用或防火墙阻止

**解决方案**:
```bash
# 查看端口占用情况
netstat -anp | grep 32896

# 修改程序中的UDP_PORT配置（需改一个不被占用的端口）
# 修改 UDP_PORT = 32896 为其他端口，如 UDP_PORT = 32897
```

### 问题2: "丢包率过高" (>5%)

**原因**: 
- UDP缓冲区太小
- 网络拥塞
- 接收机CPU占用过高

**解决方案**:
```bash
# 增加系统UDP缓冲区
sudo sysctl -w net.core.rmem_max=134217728
sudo sysctl -w net.core.rmem_default=134217728

# 检查CPU核心绑定是否成功
taskset -p -c $$  # 查看当前进程绑定的核心

# 运行高级版本以获得更好的统计
python3 udp_verifier_advanced.py
```

### 问题3: "视频加载失败" 或 "无法找到参考数据"

**原因**: 参考文件路径不对

**解决方案**:
```bash
# 检查文件是否存在
ls -l cycle/_AKBK0.txt
ls -l cycle/video10.mp4

# 如果路径不同，修改程序中的路径配置
RADAR_TXT_PATH = 'cycle/_AKBK0.txt'
VIDEO_PATH = 'cycle/video10.mp4'
```

### 问题4: "解析错误" 或 "验证数据异常"

**原因**: 数据格式与预期不符

**解决方案**:
```bash
# 查看日志中的详细错误信息
grep "ERROR\|WARN" udp_verify_*.log

# 检查FPGA发送的数据包格式是否与udp_receiver.py一致
# 或联系FPGA开发人员确认数据格式
```

## 数据格式说明

### UDP数据包结构

```
[PacketHeader: 13字节] + [PayloadData: 1440字节]

PacketHeader 格式:
├── pkt_type (1字节): 0x01=图像, 0x02=雷达
├── frame_id (2字节): 帧ID (大端序)
├── timestamp (8字节): 时间戳 (大端序)
└── sub_id (2字节): 子包ID (大端序)
```

### 图像传输

```
总共640个图像子包 (sub_id: 0-639)
├── 每包1440字节的RGB像素数据
├── 640×480×3 = 921,600字节
├── 需要640个包 = 每包1440字节
└── 帧间隔：50ms (20fps)
```

### 雷达数据传输

```
1个雷达子包 (sub_id: 640)
├── 前4字节: 目标数量 (小端序 uint32)
├── 后续数据: 最多64个目标
│   └── 每个目标16字节: id(u16) + r(s16)/100 + v(s16)/100 + a(s16)/100 + p(u16)/100 + 填充
└── 剩余填充0x00
```
